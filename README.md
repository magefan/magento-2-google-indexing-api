## Features
### Send Google Indexing Requests Through API

We‚Äôve all been used to the ‚Äúusual‚Äù way of Google crawling and discovering pages through the sitemap. While reliable, it is not the fastest way of submitting pages for Google indexing. Google might miss some pages or simply take a while to discover them.

Google Indexing API compensates for that. You can use the Magento 2 Google Indexing Extension to send indexing requests instantly, manually or automatically. It will prompt Google to schedule a crawl and reflect your changes in search results almost immediately.

  <img width="860" height="428" src="https://cm.magefan.com/wysiwyg/automatic-google-indexing-magento.png">
</p>

### Request Google Indexing for Multiple Pages

Remember Google Search Console and sending indexing requests there? It takes hours to submit each page for indexing one by one. And you still have to wait for the requests to be scheduled, let alone executed. 

With the Magento 2 Google Indexing API extension you can send multiple indexing requests in one click. Simply paste all the URLs you‚Äôd like to send for indexing and press the ‚ÄúSubmit‚Äù button.

It is that simple.

<p align="center">
  <img width="860" height="428" src="https://cm.magefan.com/wysiwyg/request-google-index-magento.png">
</p>

### Remove pages from Google Index

Sometimes it happens that you remove a page from the sitemap and set NOINDEX NOFOLLOW robots for it, but Google still somehow finds it. Correspondingly, the page still appears in search results. 

You can submit a page for removal to Google Search Console. But it will be removed for 6 months only. To remove a page from Google Index, use the Magento 2 Google Indexing Extension. Submit one or multiple URLs for removal within seconds.

<p align="center">
  <img width="860" height="428" src="https://cm.magefan.com/wysiwyg/delete-pages-from-google-index.png">
</p>

### Monitor Indexing Requests in Logs

To make sure each request, indexing or removal, you send is processed accordingly, check the logs. The Magento Google Indexing API module comes with the schedule log where you can find all indexing requests you send. 

Filter them by creating/scheduling/executing date, URL, status and action and monitor your Google indexing requests better.
